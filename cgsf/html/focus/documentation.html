01. About Goddess

Goddess - is a one page website template with a modern design based on Bootstrap 3. Goddess is supported by all modern browsers, including mobile devices. The template has responsive design and graphic optimized for high-resolution screens (retina-ready). Goddess has five landing pages: bootstrap slider, Revolution Slider, Vimeo, YouTube and Rainy Landings. The template consists of modules that can be easily deleted, added or exchanged. The distinction of Goddness is the switch button for touch devices. With the help of this button you can activate all the hover effects (which is very convenient for touch devices). The template menu is implemented in the manner of the sidebar menu of the Google Nexus 7 page. Besides Goddess has two variants of the main page with a loopable video (YouTube and Vimeo API).
The template has the main slider with a unique design, svg-animation, font Awesome icons, pricing tables, blog with topics, simple portfolio with integrated FancyBox plugin, Google Maps and Google Street View. The template also has two working php-forms with validation (feedback and subscription). Goddess already comes with color schemes.

03. Template's elements

03.01 Button for touch-devices

This button is designed for convenient work on touch devices. It can be also used for other purposes (for example to always see the blog title). The button activates all the hover effects on the page, so that the user will never miss anything important.

General principles (for developers)

".Not-global-detect" class is set for the body by default. This class is used for the landing slider only. It is responsible for the work of chevron-slider-landing. Once you press the button, ".not-global-detect" class is deleted forever. Afterward you start working with two classes: "global-mobile-on" and "global-mobile-off". Further switching between them will affect the landing slider the following way: the chevron either always visible or completely invisible. (all the hover-effects are disabled in these classes).

03.02 Video

The template has two additional versions with a video cutscene - YouTube and Vimeo. The video starts automatically and plays cyclic. In the upper right part of the screen the video has its own control buttons. Vimeo API allows you to control the video online only. Each video has an invisible layer preventing clicks. You can disable it by deleting div class="iframe-block". Video control buttons are only visible when a user brings the mouse over the video. When the button for touch devices is active, video control buttons are visible all the time.

Play Pause Stop Mute Sound

chevron-slider-landing

Chevron-slider-landing is always displayed over the slider. When a user brings the mouse over the slider, chevron is hidden, so the user can watch the slides. Thus the chevron is visible in three following cases: when the cursor is out of the slider, when the cursor is in the upper part of the slider or when the touch devices button is activated. All the logic is developed in CSS.

How to insert a video from YouTube:

Use the video-youtube.html template, optimized for YouTube. Find "Wrapper for video" and replace the link. The video cycles through the playlist.

  • - fitvids.js
  • - video-youtube.js
  • - video html-controls only for youtube

    Elements used:

03.04 Sliders (Carousels)

The template has three carousels. The first one (Landing slider) is for large full-size photos. The second slider is perfect for images with transparent background. The third one shows you the way to rotate multiple elements (Our clients). In the landing slider all the navigation buttons are created with the use of borders and have different (not mirror-like) settings. Make sure to have a good understanding of layout settings if you decide to change them.
Using Bootstrap carousel you can create a prototype of a navigation page with any section of the website taking "Our Clients" section as an example.
More detailed information on how to work with Bootstrap carousel can be found here.

  • chevron-slider-core — element, helping to position all the other included chevrons to build a background. Used in the slider #2.

03.08 Fancybox 2 plugin (Portfolio)

Portfolio is represented as a simple section with constant-width miniatures (do not depend on the screen width). On 320 screen resolution mobile phones the images are cut. You can only insert images, video or other type of content with preview to this portfolio. It doesn't have navigation, filter and sorting. However you can easily implement the pagination by means of Bootstrap carousel.
Previews have two color styles: the theme color and black. To use the theme color, please apply .portfolio-thumb-svg style. To use black color, apply .portfolio-thumb-svg-invert style.
FancyBox 2 plugin is used as the content preview. The styles and the settings for this plugin can be found in the following file: fancybox-theme.css. To change the preview background, open common.js and find "// Initialize Fancybox". Change the background color 'background' : 'rgba(255, 255, 255, 0.9)'. All the details on the plugin can be found here.

03.09 Pricing Tables

Pricing tables are created with .col-* styles. To make all the columns of the same height without using table styles in css, the parent has .pricing-* style with the number showing the maximum quantaty of positions in a table. E.g. if 6 is the maximum number of positions, apply .pricing-6 style to the parent. By default the maximum number of positions supported is 10, but you can make changes to the styles and extend the number of positions.
When mouse is over one of the positions - the equivalent positions in the neighbor tables are highlighted. It is implemented through the use of JS (in the common.js). By default the code is written for the first 10 positions, however it can be modified and the number can be increased.

03.10 Blog

The blog consists of image previews. The title and additional information is displayed when mouse over or when the button for hover effects is activated. The animation used for the blog is created by means of svg. For that reason it takes some effort to customize the internal sections, so they have a lot of requirements in the file theme-responsive.css. For the same reason the following background image size is highly recommended — 1800x500. Hover effects also have two styles: theme color and black color. Style .blog-topic-svg is used for the first case, for the second case - .blog-topic-svg-invert.
All the topics open in a new window and have internal navigation between themselves and the top of the screen.

03.19 Images. Recommendations

Landing slider

Any large image will work for the main slider. It is highly recommended to use images of the same size in order to avoid the slider jumps.

About Us

This section can be filled with photos of any size and proportions, however, it is highly recommended to use images of the same size in order to avoid the slider jumps.

Portfolio

In this section thumbnails have a strictly required size. The proportions of the images are 10:4. Any size of this proportion will fit. The recommended sizes for the images are: 350x140 (700x280 for retina).

Blog

The topics preview have svg-animation, where information can not be easily customized. For that reason they have a lot of requirements, such that it is highly recommended to use images of 1080х500px size or larger.

Contacts

Any image size will work here. The recommended sizes are: 240x150 (480х300 for retina).

Images inside text

  • class="img-wrapper-right" — wrapper for aligning to the right. It has maximum width of 50%. On small size screens stretches up to 100% width.
  • class="img-wrapper-left" — wrapper for left side alignment. It has maximum width of 50%. On small size screens stretches up to 100% width.
  • class="img-topic-full" — applied directly to the image, contains spaces. Recommended to be used with .img-responsive class.

04. Help & FAQ

04.01 Will embedded Vimeo videos play on mobile devices?

Latest mobile devices can play Vimeo’s SD and HD videos.
Plus and PRO members can create mobile versions of their videos from their Global Video Settings. These extra-small videos are compatible with lots of older mobile devices that can’t handle our SD or HD files.

04.02 Can I make my embedded vimeo videos default to HD?

Plus and PRO members have the ability to not only embed their videos in glorious HD, but also to default those videos to be viewed in HD. This is a sweet way to make sure that your crisp, high definition videos are seen in their full glory when they're embedded around the interwebs.
If you'd like all of your videos to default to HD, just go to your Account Settings and select the Videos tab. Under "What about source files, mobile, and HD embedding?" select "Always play my videos in HD, when possible" and save. Check "Also, apply to existing videos" to do just that.
Otherwise, if you have a specific video you want to default to HD mode, head to that video page and click “Settings.” From there, select the Embed tab. Check the box next to "Default this video to HD quality when embedded" and save it like your life depends on it.
With this option turned on, the only situation in which your videos will not play in HD is when your viewer manually turns HD off.

04.03 Video controls for Vimeo do not work

Video controls for Vimeo work online only. Locally they do not have actions.

04.04 How to change the background for Fancybox overlay?

Open common.js and find "// Initialize Fancybox". Change the background color 

'background' : 'rgba(255, 255, 255, 0.9)'

04.05 How to change the color of the buttons in Fancybox?

The template goes with a PSD-file fancybox_sprite.psd (in the folder «psd»). Open it in Photoshop and create any color for the buttons. If you are unable to do it yourself, you can choose the most suitable color from the suggested: (images/fancybox/controls-themes) or choose black.

04.06 Mousewheel scrolling doesn't work in Fancybox

Mousewheel scrolling not work when navigation arrows are outside of fancybox popup. Learn more about this issue

04.07 How to change Google Map marker?

Open common.js and find "// Initialize Google Map". In the section "// Marker" change the icon "fa-map-marker": 

<div class="gmap-overlay-marker"><i class="fa fa-map-marker"></i></div>
To change the color or size please edit style ".gmap-overlay-marker i" in theme.css.

04.08 How to change slide transition mode in Bootstrap carousel?

The slider supports two types of animation: fade and slide.
By default slide transition goes with horizontal slip. To change the animation, add class .carousel-fade to the main div.

04.10 Contact form doesn't work?

Both forms are developed in php. To make them work one needs a php-server.
If the contact form shows an error, it's most probably an issue with the file path. Try to delete slash from here: html form (action="/php/feedback.php"). Here slash directs to the root catalog. The form will not work also if index.html is not in the root folder.

04.11 How to change the email address for the contact form?

Open "contact.php" file and insert your email address here: 

$emailTo = 'your@mail.com';

04.12 Video doesn't work in Fancybox?

If you use Fancybox media helpers for the video, you'll need a server for that. That might be the reason why video is not found.

04.13 Advertising on a YouTube video shows up

Ads can be blocked from the video author's account only.

04.14 Some images are hidden with attribute @2x?

retina.js has a issue. Script make width and height of hidden image equal 0. Template goes with a corrected file retina-new.js.

04.15 Video doesn't play on iOS devices?

On iOS, playback can only be initiated from a user action. When calling play with the API, somewhere along the line that user action is lost, and triggering the actual video to play doesn't work. We've tried to find a way around it, but so far nothing we've tried has worked.

04.16 SVG don't change color depending on the color schemes in the Revolution slider

Color is written in the html file, not in the css.

04.17 How to mute the landing video (vimeo/youtube)?

Muting Vimeo

In the index-vimeo.html find after tag "body" this script and change number to "0":

$('.video-wrapper').fitVids();

(function(window) {
    window.vimeo_player_loaded = function (player){
        player.api('setVolume', 0.5); 
        player.api('play');
    }
})(window);

Muting YouTube

In the index-youtube.html find after tag "body" this script and UNcomment selected text:

$(function(){
    $('.video-wrapper').fitVids();
});

(function (playerId) {
    var tag = document.createElement('script');
    tag.src = "http://www.youtube.com/player_api";
    var firstScriptTag = document.getElementsByTagName('script')[0];
    firstScriptTag.parentNode.insertBefore(tag, firstScriptTag);
    window.onYouTubePlayerAPIReady = function() {
        new YT.Player(playerId, {
            events: {
                'onReady': function (event) {
                    event.target.mute();
                }
            }
        });
    };
})('yt-player'); // Youtube Iframe Id

04.18 How to change background image for rainy landing?

In the index-rainy.html find after tag "body" this script and change selected image:

Note: some paths couldn't works properly. For more information see documentation for the plugin

$(function(){
    var image = document.getElementById('rainy-image'),
        parentElement = document.getElementById('cholder');

    image.onload = function(){
        var engine =  new RainyDay({
            'image': this,
            'parentElement': parentElement
        });
        engine.trail = engine.TRAIL_DROPS;
        engine.gravity = engine.GRAVITY_NON_LINEAR;
        engine.rain([
            [1, 0, 20],
            [3, 3, 1]
        ], 100);
    };
    image.crossOrigin = 'anonymous';
    image.src = '/images/your_image.jpg';
})
